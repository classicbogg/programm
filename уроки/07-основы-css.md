## Главное

- Стили в **отдельном файле**, подключение через `<link>` в `<head>`.
- Стилизуй в основном через **классы**, не через теги везде подряд.

### Специфичность (микро-версия)

Если два правила спорят, обычно выигрывает более точный селектор.

```css
p { color: #111; }
.card p { color: #2563eb; } /* это правило применится внутри .card */
```

Практика: меньше длинных цепочек, больше простых классов.

### Именование классов

| Хорошо | Плохо |
|-----------|----------|
| `.site-header` | `.div1` |
| `.nav` | `.block` |
| `.card` | `.red` |

Правило: **имя = роль блока**, не цвет и не номер.  
Шпаргалка: [шпаргалки/ШПАРГАЛКА-CSS.md](../шпаргалки/ШПАРГАЛКА-CSS.md).

**Emmet:** в HTML — `link:css` + Tab для подключения стилей; остальные теги — через подсказки (`header`, `nav`).

```html
<link rel="stylesheet" href="css/styles.css">
```

```css
* {
  box-sizing: border-box;
}

body {
  margin: 0;
  font-family: system-ui, sans-serif;
}

.nav a {
  text-decoration: none;
}
```

## Типичные ошибки урока

1. CSS подключён не в `<head>` или путь к файлу неверный — стили не применятся.
2. Стили пишут в `style=""` в HTML вместо отдельного `styles.css`.
3. Классы вроде `.div1`, `.block2` — переименуй по роли блока.
