## 💡 Главное

- Стили в **отдельном файле**, подключение через `<link>` в `<head>`.
- Стилизуй в основном через **классы**, не через теги везде подряд.

### Специфичность (микро-версия)

Если два правила спорят, обычно выигрывает более точный селектор.

```css
p { color: #111; }
.card p { color: #2563eb; } /* это правило применится внутри .card */
```

Практика: меньше длинных цепочек, больше простых классов.

### Именование классов

| ✅ Хорошо | ❌ Плохо |
|-----------|----------|
| `.site-header` | `.div1` |
| `.nav` | `.block` |
| `.card` | `.red` |

Правило: **имя = роль блока**, не цвет и не номер.  
Шпаргалка: [ШПАРГАЛКА.md](../ШПАРГАЛКА.md).

**Emmet:** в HTML `header.site-header` + Tab; в CSS `link:css` + Tab в `<head>`.

```html
<link rel="stylesheet" href="css/styles.css">
```

```css
* {
  box-sizing: border-box;
}

body {
  margin: 0;
  font-family: system-ui, sans-serif;
}

.nav a {
  text-decoration: none;
}
```

## ✍️ Практика

1. Создай `css/styles.css`, подключи на обеих страницах.
2. Добавь классы в HTML: `site-header`, `nav`, `site-footer`.
3. Убери подчёркивание у ссылок меню, фон у `body`.
