## ✍️ Практика

Доведи `my-site/` до финала **без CSS**.

**Правило урока:** сначала чистая и осмысленная HTML-структура, потом CSS.  
Если структура "плавает", стили только замаскируют проблему.

### `index.html`

- [ ] Семантика: header, nav, main, section, footer
- [ ] h1, секции с h2
- [ ] Текст, список навыков, фото с alt
- [ ] Ссылки в контактах
- [ ] Таблица (данные, не вёрстка)

### `contact.html`

- [ ] Та же шапка и подвал
- [ ] Форма: label, required, select, textarea

## Проверка разметки

Прогони `index.html` через [валидатор HTML](https://validator.w3.org/):

1. Open the Web Page → укажи файл или URL Live Server.
2. Исправь **ошибки** (warnings можно оставить на этом этапе).

## 🔎 Микро-ревью (необязательно, но полезно)

Если хочешь закрепить результат перед CSS-блоком, сделай короткую самопроверку:

1. Прочитай ещё раз [ЧЕКЛИСТ.md](../practice/ЧЕКЛИСТ.md) для HTML
2. Открой `index.html` и `contact.html` в браузере и пройди глазами структуру (`header`, `main`, `footer`, форма)
3. Попроси одногруппника быстро проверить 3 пункта:
   - один `h1` на странице
   - есть `alt` у изображений
   - у полей формы есть `label`

Это не обязательно, но после такого ревью на CSS переходить заметно легче.
