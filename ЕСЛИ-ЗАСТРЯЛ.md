# Troubleshooting для учеников

Быстрый разбор типичных проблем на курсе.

---

## 1) CSS не применяется

Проверь:

- Файл точно называется `styles.css` (без `styles.css.txt`)
- В `head` есть `<link rel="stylesheet" href="css/styles.css">`
- Путь к файлу верный относительно HTML
- Файл сохранен (`Ctrl+S`)

---

## 2) Картинка не отображается

Проверь:

- Файл лежит в нужной папке (`images/`)
- Путь в `src` совпадает с именем файла
- Регистр букв совпадает (`Photo.jpg` не равно `photo.jpg`)
- Формат поддерживается (`.jpg`, `.png`, `.webp`)

---

## 3) Ссылка не открывается

Проверь:

- Для внешних ссылок есть `https://`
- Для внутренних путь правильный (`contact.html`)
- Используется `href`, а не `src`

---

## 4) Emmet не разворачивается

Проверь:

- Курсор стоит в текстовом контексте (не в комментарии)
- Нажимаешь `Tab` (или `Enter`, если Tab переопределен)
- Файл имеет правильное расширение (`.html`, `.css`)

---

## 5) Flexbox "не работает"

Чаще всего причина:

- `display: flex` поставлен не на родителя, а на дочерний элемент
- Перепутаны `justify-content` и `align-items`
- Нет `flex-wrap`, поэтому карточки не переносятся

---

## 6) Ломается на телефоне (320px)

Проверь:

- Есть `meta name="viewport"` в `head`
- Картинки с `max-width: 100%`
- Нет фиксированных широких блоков (`width: 1200px`)
- Проверил `@media (min-width: 768px)` и mobile-first стили

---

## 7) Форма неудобна или невалидна

Проверь:

- У каждого поля есть `label`
- `for` у label совпадает с `id` поля
- Обязательные поля отмечены `required`
- `placeholder` не заменяет `label`

---

## 8) В валидаторе ошибки

Проверь типовые причины:

- Незакрытые теги
- Вложенность сломана (`li` вне `ul/ol`)
- Дублирующиеся `id`
- Отсутствуют обязательные атрибуты

Исправляй сначала ошибки, потом предупреждения.

---

## Когда просить помощь

Если застрял дольше 20 минут:

1. Отправь наставнику файл + описание проблемы
2. Напиши, что уже пробовал
3. Покажи сообщение ошибки (если есть)

Не отправляй только скрин без кода.
