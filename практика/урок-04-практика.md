---

**Ориентир по времени:** 50–60 минут.

**Цель:** перестроить страницу на семантические теги вместо «голых» `div`.

**Контекст:** из текстового черновика делаешь структурно правильную визитку. `resume.html` — вторая учебная страница: на ней тренируешь ту же схему разметки.

## База

1. Перестрой `index.html` по схеме: `header` → `main` → `footer`.
2. В `header` размести имя/логотип и `nav`.
3. Внутри `main` сделай минимум 2 `section` с заголовками `h2`.
4. Проверь, что `main` на странице только один.
5. В `nav` оставь рабочие ссылки (главная, `resume.html`, якорь на секцию).

## Средний

1. Одну карточку контента оформи через `article` внутри `section`.
2. Добавь классы по роли блоков (`site-header`, `about`, `hobby`), не по цвету.
3. Используй `div` только там, где нет подходящего семантического тега.
4. Добавь `id` для якорной навигации (`id="about"`).
5. Проверь дерево в DevTools: структура должна читаться по ролям.

## Продвинутый

1. Приведи `resume.html` к той же семантической схеме.
2. Сравни обе страницы: шапка и подвал должны быть согласованы.
3. Убери лишние обёртки `div`, которые дублируют семантику.
4. Прогони `index.html` через валидатор и исправь ошибки структуры.
5. В комментарии объясни на своём примере, чем `section` отличается от `div`.

## Самопроверка

- [ ] Есть `header`, `nav`, `main`, `footer`
- [ ] `main` один
- [ ] Есть `section` и `article`
- [ ] Классы названы по роли, не по внешнему виду
