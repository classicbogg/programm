**Ориентир по времени:** 45–55 минут.

**Цель:** стабилизировать размеры блоков, отступы и ширину контента.

**Контекст:** визуал есть, но без нормальной блочной модели сайт «плывёт». Исправляешь это системно.

## База

1. Добавь в начало CSS: `* { box-sizing: border-box; }`
2. Сделай контейнер контента: `max-width` + `margin: 0 auto`.
3. Задай единые вертикальные отступы между секциями.
4. Проверь, что контент не прилипает к краям экрана.

## Средний

1. Оформи карточки: `padding`, `border`, `margin` без скачков высоты.
2. Для изображений добавь `max-width: 100%`.
3. Проверь таблицу и форму: они не должны вылезать за экран.
4. Убери фиксированные ширины, которые ломают адаптив.
5. Сравни блоки до и после — отступы должны стать предсказуемыми.

## Продвинутый

1. Найди 3 места с переполнением (длинные слова, картинки, поля формы).
2. Исправь переполнение без «костылей» вроде `overflow: hidden` на всём сайте.
3. Выровняй внутренние отступы карточек по единому ритму (например, 16/24px).
4. Проверь страницу на ширине 320px и 1200px.
5. Зафиксируй в комментарии CSS, какие правила box model ты используешь как стандарт проекта.

## Самопроверка

- [ ] `box-sizing: border-box` включён
- [ ] Контейнер центрируется и ограничен по ширине
- [ ] Карточки и секции имеют ровные отступы
- [ ] Нет горизонтального скролла из-за блоков
