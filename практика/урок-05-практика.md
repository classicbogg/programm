# Практика к уроку 5: формы

---

**Ориентир по времени:** 60–75 минут.

**Цель:** собрать рабочую форму с базовой доступностью и таблицу с данными.

**Контекст:** создай `contact.html` как третью полноценную страницу сайта. Шапка, подвал и **одинаковое меню** — на `index.html`, `resume.html` и `contact.html`.

## База

1. Создай `contact.html` с той же семантической структурой (`header`, `main`, `footer`), что на главной.
2. На **всех трёх** страницах выровняй `nav`: главная (`index.html`), резюме (`resume.html`), контакты (`contact.html`).
3. Собери форму обратной связи из 7 полей: имя, email, телефон, тема, тип обращения (radio), чекбокс согласия, сообщение.
4. Добавь на `index.html` таблицу с данными (расписание, цены, сравнение — не для вёрстки страницы).
5. Проверь переходы между всеми тремя страницами через меню.

## Средний

1. Для каждого поля формы обеспечь связку `label` + `id`.
2. Добавь `required` и `placeholder` там, где это действительно помогает пользователю.
3. У кнопки отправки укажи `type="submit"` явно.
4. Добавь `select` с несколькими вариантами темы обращения.
5. Проверь, что `placeholder` не заменяет `label`.

## Продвинутый

1. Протестируй форму с клавиатуры (Tab / Shift+Tab).
2. Исправь порядок фокуса, если он «прыгает».
3. Проверь обязательные поля: форма не должна отправляться с пустыми `required`.
4. У таблицы добавь `caption` или поясняющий `h2` рядом.
5. Прогони `contact.html` через валидатор и исправь ошибки формы.

## Самопроверка

- [ ] Одинаковое меню на `index.html`, `resume.html`, `contact.html`
- [ ] У каждого поля есть `label` с правильным `for`
- [ ] Таблица только для данных, не для макета
- [ ] Навигация между всеми страницами работает
- [ ] Форма проверена с клавиатуры
