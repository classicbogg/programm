---

**Ориентир по времени:** 90–120 минут.

**Цель:** довести HTML-часть проекта до состояния «готово к CSS».

**Контекст:** контрольная точка блока HTML. Стили не добавляешь — только структура, содержание и качество разметки.

**Правило:** доведи `my-site/` до финала **без CSS**.

## База

Доведи `index.html`:

1. Семантика: `header`, `nav`, `main`, `section`, `footer`.
2. Один `h1`, секции с `h2`.
3. Текст, списки «Что умею» и «Хобби», фото с `alt`.
4. Блок контактов со ссылками.
5. Таблица с данными (не для вёрстки страницы).

Доведи `contact.html`:

1. Та же шапка и подвал, что на главной.
2. Форма с `label`, `required`, `select`, `textarea`.

Доведи `resume.html`:

1. Та же шапка, подвал и меню, что на главной.
2. Осмысленный контент в `main` (не пустая страница).

## Средний

1. Открой страницы через Live Server (правый клик по `index.html` → **Open with Live Server**).
2. Прогони все три файла через [валидатор HTML](https://validator.w3.org/) (Open the Web Page → файл или URL Live Server).
3. Исправь все ошибки (warnings — только если понимаешь причину).
4. Проверь навигацию из всех пунктов меню (`index.html`, `contact.html`, `resume.html`).
5. Проверь якорные ссылки и пути к изображениям.

## Продвинутый

1. В DevTools (F12 → **Elements**) найди в дереве `form` и `nav`.
2. Проведи аудит доступности: `alt`, `label`, понятные ссылки, Tab-порядок.
3. Выпиши 5 проблем, которые нашёл, и как исправил каждую.
4. Пройди [ЧЕКЛИСТ.md](./ЧЕКЛИСТ.md) для HTML-блока.
5. Попроси одногруппника проверить 3 пункта и исправь замечания.

## Самопроверка

- [ ] Три страницы полностью рабочие и с одинаковым меню
- [ ] Валидатор без критичных ошибок
- [ ] HTML-часть чек-листа закрыта
- [ ] Проект готов к подключению стилей
